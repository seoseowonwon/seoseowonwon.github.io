클릭 시 프로젝트로 이동
파이널 프로젝트

영화 예매 사이트 2024-07-09 ~ 2024-08-26

프로젝트 개요 배운 점
제목 : #CINEMA
기간 : 2024년 07월 09일 ~ 2024년 08월 26일
인원 : 1명


 첫 번째, 프로젝트를 진행하면서 문제에 봉착해 여러 번 초기 상태로 되돌아가게 되는 경험을 통해, 프로젝트의 계획과 기획 단계 에서 철저한 설계의 중요성을 깨달았습니다.

예를 들어, API와 데이터베이스 설계를 명확히 하지 않고 작업을 시작하면, 후에 데이터 모델이 변경되거나 API 응답 형식이 맞지 않아 큰 수정 작업이 필요하게 되었고, 따라서 프로젝트 초기 단계에서 요구 사항을 명확히 하고, 시스템의 구조와 데이터 흐름을 잘 설계하는 것이 성공적인 프로젝트 수행에 큰 도움이 되는 것을 배웠습니다.


두 번째, API와 AJAX를 이용해 서버와 클라이언트 간의 비동기 통신을 구현하면서, 클라이언트 측에서 서버와의 실시간 데이터 교환의 중요성을 배웠습니다.

예를 들어, 사용자가 날짜를 선택했을 때, AJAX를 통해 서버에 요청을 보내고 실시간으로 극장 정보를 업데이트하는 기능을 구현하면서, 사용자 경험을 향상시키는 방법을 익혔습니다.


세 번째, 프로젝트 개발 중 여러 오류와 문제에 직면하면서, 문제 해결과 디버깅의 중요성을 깊이 이해하게 되었습니다.

예를 들어, AJAX 요청이 실패하거나 API 응답이 예상과 다를 때, 브라우저의 개발자 도구를 활용하여 요청과 응답을 분석하고 문제의 원인을 파악하는 과정에서 디버깅 기술이 얼마나 중요한지를 배웠습니다.

담당기능
  1. 로그인 페이지 구현
    • - MariaDB에 저장된 ID와 비밀번호를 비교하여, 일치할 경우 로그인되도록 구현함.
  2. 영화 선택 페이지
    • -KOBIS API를 통해 박스오피스 순위별로 영화 목록을 출력함.
      -영화를 선택하면, 미리 정의된 여러 지역, 영화관 이름, 날짜, 시간이 임의로 MariaDB에 배정됨.
      -영화 선택 시, 지역 선택 버튼 -> 영화관 선택 버튼 -> 오늘 날짜부터 11일 후까지의 날짜 선택 버튼 -> 시간 선택 버튼이 순차적으로 출력되도록 구현함.
      -시간 선택 버튼을 누를 시, 영화명, 영화관명, 예약 날짜, 시간을 form의 POST 방식으로 좌석 예매 페이지로 전달함.
  3. 좌석 예매 페이지
    • -좌석이 이미 예매되었는지 DB에서 확인한 후, 예약된 좌석은 클릭할 수 없도록 하고, 예약되지 않은 좌석은 선택할 수 있도록 구현함.
      -좌석 선택 시, 청소년과 일반 관람객을 구분하여 최대 8명까지 선택할 수 있도록 하며, 총 인원이 8명을 초과할 경우 다시 인원을 선택하도록 함.
      -이상이 없을 경우, 결제 페이지로 이동하도록 구현함.
  4. 결제 페이지
    • -선택된 좌석과 성인 11000원, 청소년 8000원으로 총 결제 금액을 표시함.
      -결제 후 이상이 없을 경우, 영화 선택 페이지로 이동하도록 구현함.

대학교 LMS(학습 관리 시스템) 2024-05-09 ~ 2024-06-12

프로젝트 개요 배운 점
제목 : 대학교 LMS(학습 관리 시스템)
기간 : 2024년 05월 09일 ~ 2024년 06월 12일
인원 : 4명
 이 프로젝트를 통해 소통과 분배의 중요성을 크게 느꼈습니다.

이 프로젝트는 제 첫 팀 프로젝트였고, 네 명이서 역할을 분담하는 과정에서 예상치 못한 어려움들이 있었습니다.
저희 팀은 기획 작업을 통해 문서 작성 및 기획에 대한 이해를 높였습니다. 그러나 이 과정에서 필요한 추가 기능들이 많이 나타났습니다.
이를 통해 처음부터 완벽한 기획은 어렵다는 것을 깨달았고, 추가 기능에 대한 일정을 미리 고려해야 함을 인지했습니다.

특히, 제가 담당한 학생 관리 시스템에서는 변수가 많이 발생했고, 이로 인해 추가 기능이 계속해서 추가되었습니다.
이러한 문제를 팀원들과의 소통을 통해 해결할 수 있었습니다.

저는 미리 역할 분배를 잘 조정하여, 팀원들이 저의 담당 파트 중 일부를 맡을 수 있도록 했습니다.
이로 인해 팀워크가 잘 이루어졌고, 함께 문제를 해결하면서 즐거운 경험을 할 수 있었습니다.

이 경험을 통해, 저의 인지적 능력과 자기 파악 능력이 팀 프로젝트에 긍정적인 영향을 미칠 것이라고 자신합니다.
이는 앞으로 회사에서도 신입 개발자로서 중요한 덕목이 될 것이라 생각합니다.
담당기능
마이 페이지 수정, 비밀번호 변경, 목록 페이지, 상세보기 페이지, 수강신청 페이지, 수강신청 장바구니, 나의 강의 페이지, 공지사항 페이지, 학생 성적 페이지 등 CRUD

  1. 관리자 페이지
    • 마이페이지(관리자)
      - 관리자 번호, 이름, 성별, 생일, 연락처, 집 주소, 이메일, 등급 표시
      - 이메일 주소, 연락처, 집 주소, 성별, 생일등을 수정할 수 있도록 함
    • 관리자 목록
      - 관리자가 마스터 등급일 경우(if(!grade.equals("마스터"))), 등록된 관리자들을 리스트 형태로 표시
      - 특정 관리자 클릭 시, 해당 관리자의 상세정보 표시
      - 목록 옆의 삭제 버튼을 눌러 관리자를 삭제할 수 있음
    • 관리자 등록 페이지
      - 마스터 등급의 관리자일 경우, 관리자 등록란이 생성되고 관리자 등록 가능하도록 함
    • 관리자 비밀번호 변경
      - 만약 새로 변경할 비밀번호가, 어드민 비밀번호 기록(admin_pw_history)중 최근 3개의 비밀번호와 일치한다면 변경할 수 없음
      - 위의 조건에 해당되지 않는다면, 비밀번호 변경 후 admin_pw_history에 비밀번호 추가
  2. 학생 페이지
    • 수강신청 목록
      - 관리자에의해 승인된 수강신청 목록들이 표시됨. 이를 학생이 수강 신청할 수 있도록 함
    • 수강신청 장바구니
      - 5과목에서 ~ 8과목을 수강할 수 있으며, 이에 해당하지 않을 시 수강신청이 안되도록 구현
    • 강의 목록
      - 내가 수업들어야할 강의들의 목록 표시
      - 클릭시 해당 강의 상세정보 표시
  3. 교수 페이지
    • 교수 강의 목록
      - 자신의 강의들이 목록 형태로 표시
      - 클릭시 해당 강의 상세 정보 표시
    • 공지사항 목록
      - 관리자가 등록한 공지사항목록을 조회
      - 클릭시 해당 공지사항 상세 정보 표시

성격의 장단점


첫째는 분석적입니다. 저는 어떤 일을 왜 잘하게 되었는지, 그 과정에서 어떤 요소들이 중요했는지를 항상 고민하고 분석하는 습관을 지니고 있습니다. 예로 들면, 제가 뭔가를 평소보다 더 잘 수행한 날이 있을 때, 그 원인을 분석합니다. 잘된 날이 있을 때, 평소와 어느 점이 달랐는지 등을 생각하고 분석하여 더 좋은 결과를 도출할 수 있도록 합니다. 코딩할 때도 각 코드를 리뷰하며 더 편리한 코드는 없었는지 등을 분석하곤 합니다. 이는 제가 개인적인 발전과 업무에서의 성과를 극대화하기 위한 중요한 장점이라고 생각합니다.

둘째는 책임감입니다. 저는 책임감을 매우 중요하게 생각합니다. 저는 어떤 상황에서도 제 책임을 완수하려고 노력하고 있습니다. 그래서 친구들과의 약속이든, 업무상의 약속이든, 저는 상대방에게 신뢰를 지키기 위해 항상 약속을 지키려고 노력하고, 아르바이트하면서, 근무 일정에 맞춰 한 번도 결근한 적이 없는 경험을 통해 책임감을 보여주었습니다. 이런 책임감을 바탕으로 저는 어떤 일이든지 끝까지 책임을 질 수 있음을 자부할 수 있습니다.


단점은 아쉬운 점이 남으면 놓지 못하고 계속 얽매인다는 점입니다. 그러나 이러한 점은 대충 문제를 넘기지 않고 성과 달성을 위한 중요한 원동력이 된다고 생각합니다.




신입 개발자로서 이러한 성격과 경험들을 바탕으로 회사와 팀에 부드러운 협업 분위기와 기록을 제공하고자 합니다. 계획을 세우고 우선순위를 파악하여 업무를 처리하며, 원활한 소통 능력으로 동료들과 효과적인 협력을 지향합니다. 또한 다양한 활동을 추구하며 건강한 사내 문화와 팀원들과의 유대감, 긍정적인 동료 관계 구축에 기여하고자 합니다. 이러한 성격과 노력을 바탕으로 회사의 목표 달성과 성장에 기여하는 개발자가 되고 싶습니다.
